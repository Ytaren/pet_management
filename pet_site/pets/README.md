# 🐾 Pets 应用 - 宠物档案管理系统

## 📋 应用概述
`pets` 应用负责管理宠物档案系统，包括宠物的基本信息、个性描述、医疗备注等核心数据。该应用已优化，移除了芯片功能，专注于核心宠物信息管理。

## 🏗️ 应用架构 (已优化)
```
pets/
├── __init__.py              # 应用初始化
├── README.md               # 应用说明文档 (本文件)
├── admin.py                # 后台管理配置 (已移除芯片字段)
├── apps.py                 # 应用配置
├── models.py               # 宠物数据模型
├── views.py                # 宠物视图逻辑

    └── pets/
```

## 📄 核心文件说明

### 🗃️ 数据模型 (`models.py`)
- `Pet` - 宠物基本档案模型
  - 基本信息：姓名、品种、年龄、性别、体重
  - 详细信息：个性描述(personality_notes)、医疗备注(medical_notes)
  - 照片管理：宠物照片上传
  - 时间戳：创建和更新时间

### 📝 表单管理 (`forms.py`)
- `PetForm` - 宠物信息表单 (已优化)
  - 移除了microchip_number字段
  - 包含所有必要的宠物信息字段
  - 优化的表单控件配置

## 🎯 主要功能
1. **宠物档案管理** - 创建、编辑、删除宠物档案
2. **品种信息管理** - 维护宠物品种数据库
3. **健康档案** - 记录宠物健康状况
4. **疫苗管理** - 疫苗接种记录和提醒
5. **宠物搜索** - 多条件搜索宠物档案

## 🔗 关联关系
- 与 `accounts` 应用：多对一关系（用户可拥有多个宠物）
- 与 `consultations` 应用：一对多关系（一个宠物可有多次咨询）
- 与 `logs` 应用：一对多关系（一个宠物可有多条日志记录）
- 与 `core` 应用：继承基础模型和使用共享常量

## 📊 数据模型关系图
```
User (accounts)
  ↓ 1:N
Pet (pets)
  ↓ 1:N
├── ConsultationHistory (consultations)
├── PetLog (logs)
└── VaccineRecord (pets)
```

## 🏗️ 架构特性
- **数据中心** - 作为系统的核心数据源，为其他应用提供宠物基础信息
- **模块化设计** - 与独立的consultations应用形成数据关联
- **扩展性强** - 支持品种数据库扩展和自定义字段
- **数据完整性** - 与用户、日志、咨询记录的关联保证数据一致性

## 🔧 技术实现
- 继承`core.models.TimeStampedModel`实现时间戳管理
- 使用`core.constants.PetChoices`统一宠物相关选择项
- 支持软删除和数据恢复机制
- 图片上传和处理功能

## 📊 数据结构层次
```
Pet (宠物档案表)
├── 基础信息：名称、品种、年龄、性别
├── 外观特征：颜色、体重、特殊标记
├── 健康档案：疫苗记录、过敏信息、病史
├── 关联数据：
│   ├── VaccineRecord (疫苗记录) 1:N
│   ├── PetLog (日志记录) 1:N
│   └── ConsultationHistory (咨询记录) 1:N
└── 扩展字段：照片、备注、紧急联系方式

Breed (品种表)
├── 品种信息：名称、分类、特性
├── 护理建议：饮食、运动、健康重点
└── 关联数据：Pet (宠物档案) 1:N
```

## 🤝 与其他应用的集成
- **consultations应用** - 提供宠物基础信息支撑AI咨询

- **用户友好** - 简化宠物信息录入和管理流程
