# 🔧 Core 应用 - 核心共享模块

## 📋 应用概述
`core` 应用提供项目共享的基础功能、抽象模型、通用常量和工具函数，为其他应用(accounts、pets、consultations、logs)提供统一的基础支撑。

## 🏗️ 应用架构 (基础设施)
```
core/
├── __init__.py              # 应用初始化
├── README.md               # 应用说明文档 (本文件)
├── admin.py                # 基础管理配置
├── apps.py                 # 应用配置
├── models.py               # 抽象模型和常量定义
├── views.py                # 核心视图(如果需要)
├── migrations/             # 数据库迁移文件
```

## 📄 核心功能说明

### 🗃️ 抽象模型 (`models.py`)
提供可复用的基础模型类：

#### 🕒 时间戳模型
- `TimeStampedModel` - 带时间戳的抽象模型
  - `created_at` - 创建时间
  - `updated_at` - 更新时间
  - 自动管理时间戳

#### 📊 选择常量
统一管理项目中的选择项：
- **宠物类型选择** - 猫、狗等宠物类型
- **性别选择** - 雌性、雄性、未知
- **绝育状态选择** - 已绝育、未绝育、不确定
- **咨询类型选择** - 饲养、疫苗、健康、紧急情况

#### 🔧 工具功能
- **数据验证** - 统一的验证逻辑
- **异常处理** - 自定义异常管理
- **常量管理** - 项目级别的选择常量

## 🎯 主要功能
1. **基础模型** - 提供通用的抽象模型

## 🚀 设计原则
- **DRY原则** - 避免代码重复
- **单一职责** - 每个模块职责明确
- **低耦合** - 减少模块间依赖
- **高内聚** - 相关功能集中管理
- **可复用** - 组件可在多处使用

## 🏗️ 架构地位
- **基础支撑** - 为所有应用提供共享的基础功能
- **标准统一** - 确保项目内数据结构和处理逻辑的一致性
- **扩展中心** - 新增共享功能的统一入口
- **质量保证** - 集中的验证和异常处理机制

## 🔧 核心组件详解

### 抽象模型类
- **`TimeStampedModel`** - 自动管理created_at和updated_at字段
- **`UUIDModel`** - 提供UUID主键支持
- **`SoftDeleteModel`** - 实现软删除功能，避免数据丢失

### 常量管理
- **`PetChoices`** - 宠物类型、性别、健康状态等选择项
- **`ConsultationChoices`** - 咨询类型、紧急程度等选择项
- **`LogChoices`** - 日志类型、记录分类等选择项

### 工具函数库
- **数据验证** - 统一的输入验证和格式检查
- **文件处理** - 图片压缩、格式转换等功能
- **安全工具** - 数据加密、敏感信息处理

### 异常处理
- **自定义异常类** - 业务逻辑相关的异常定义
- **错误处理机制** - 统一的错误响应和日志记录
- **安全防护** - 防止常见的安全漏洞

## 🤝 服务的应用
- **accounts应用** - 使用用户验证和权限管理组件
- **pets应用** - 继承基础模型和使用宠物相关常量
- **consultations应用** - 使用咨询相关常量和数据验证
- **logs应用** - 继承时间戳模型和使用日志类型常量

## 📈 使用统计
```
TimeStampedModel: 被4个应用的12个模型继承
PetChoices: 在pets和consultations应用中使用
ConsultationChoices: 在consultations应用中广泛使用
LogChoices: 在logs应用的多个模型中使用
工具函数: 全项目共计调用150+次
```

## 💡 维护指南
- **新增常量** - 在constants.py中统一定义
- **扩展模型** - 在models.py中添加新的抽象类
- **工具函数** - 在utils.py中添加通用功能
- **版本兼容** - 确保变更向下兼容
